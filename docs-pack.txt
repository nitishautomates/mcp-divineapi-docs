AUTH (every request)
  - Header:  Authorization: Bearer {Your Auth Token}
  - Form field: api_key={Your API Key}   (multipart/form-data on every call)
  - All requests are POST with multipart/form-data; all field values are strings.
  - Get credentials + tokens at https://developers.divineapi.com

ERROR SEMANTICS (depends on the host)
  - Legacy hosts astroapi-1 .. astroapi-5: ALWAYS return HTTP 200. Check the JSON
    body: success=1 means ok, success=2 validation error, success=3 auth error (msg field explains).
  - Structured hosts astroapi-7, astroapi-8, pdf: return real HTTP 4xx with body
    {status:"error", error_code, code, error, message}. Common:
      400 AUTHORIZATION_HEADER_MISSING (Authorization header missing / bad params)
      401 AUTHORIZATION_INVALID       (api_key or Bearer token missing / invalid)

BIRTH (the 13 standard birth params; where a card says "params: BIRTH" send all 13)
  api_key={Your API Key}  full_name=Rahul Kumar  day=24  month=05  year=2023
  hour=14  min=40  sec=43  gender=male  place=New Delhi, India
  lat=28.7041  lon=77.1025  tzone=5.5
  (Indian endpoints often need only a subset; those cards list params explicitly.)

P1_BIRTH / P2_BIRTH (two-person endpoints: synastry, composite, match-making, couple)
  The same 12 natal fields prefixed p1_ / p2_ (p1_full_name .. p1_tzone and
  p2_full_name .. p2_tzone). api_key is sent once, unprefixed.

HOROSCOPE SELECTORS (Horoscope and Tarot v2 API, host astroapi-5)
  All take sign (e.g. Aries) + tzone. The period selector differs:
    daily   -> h_day  (e.g. today; plus day, month, year)
    weekly  -> week
    monthly -> month
    yearly  -> year
  lan (2-letter language code, default en) is optional on these.

HOUSE_SYSTEM (optional on Western chart endpoints, default P)
  P=Placidus  K=Koch  O=Porphyry  R=Regiomontanus  C=Campanus  B=Alcabitius
  A=Equal  E=Equal(=A)  W=Equal/whole-sign  N=Whole sign (Aries=1st)
  M=Morinus  X=Axial/Meridian  T=Polich/Page (topocentric)

FIELD FORMATS (get these right)
  - place:  lowercase city (e.g. new delhi). Uppercase can be rejected on some hosts.
  - tzone:  decimal hours (5.5), never "+5:30" and never a timezone name.
  - sec:    required on birth endpoints; send 0 if the second is unknown (omitting fails).
  - date:   separate day, month, year fields (zero-padded strings ok), not one date string.
  - sign:   a zodiac name for horoscopes (aries .. pisces); keep it lowercase.
  - lan:    2-letter language code (en, hi, ...); optional, default en.

WHAT NOT TO DO (the common agent mistakes)
  - Do NOT send a calendar date where a keyword selector is expected
    (h_day=today|tomorrow|yesterday; week/month/year=current|prev|next).
  - Do NOT omit sec on birth endpoints (send 0).
  - Do NOT send tzone as "+5:30" or a name; use a decimal like 5.5.
  - Do NOT retry on success=3 (auth error); fix the api_key / Bearer token instead.
  - Do NOT assume v2 for the tarot / love readings; they are v3 (see the cards).
  - Do NOT invent params or response fields; each card lists exactly what exists.

RETURNS LINE
  A card's "returns:" line lists the top-level fields of the REAL response (nested
  objects shown by key name only; a repeating result shown as [list of: ...]). Use
  it to answer "does this endpoint give me X?". For full nested example bodies, see
  openapi.yaml or the Postman collection.

DOMAIN GOTCHAS (astrology-specific traps; get these right)
  - Sidereal vs tropical: Indian / Vedic endpoints are SIDEREAL, Western endpoints
    are TROPICAL. Do not carry a Western house_system onto a Vedic call.
  - tzone is a plain decimal offset (5.5 = IST) and is NOT adjusted for DST. Send the
    real offset for the birth moment, never "+5:30" or a zone name.
  - Rahu and Ketu are the lunar nodes (shadow points), not planets. Endpoints that
    take node_type accept the true vs mean node variants; choose one deliberately.
  - Horoscope timeframes are keyword selectors (today / current / prev / next),
    never calendar dates.
  - Two-person work (synastry, composite, match-making, couple reports) uses the
    p1_ / p2_ prefixed birth blocks; api_key is sent once, unprefixed.
  - house_system (Western charts) defaults to Placidus; pass a friendly name or a
    single letter (see HOUSE_SYSTEM above).
  - Legacy hosts (astroapi-1..5) return HTTP 200 even on an auth or validation
    error. Always check the body's success field (1 = ok, 2 = validation error, 3 = auth error); do not
    trust the HTTP status alone.
  - place is a lowercase city string; sec is required on birth endpoints (send 0 if
    unknown) or the call is rejected.
